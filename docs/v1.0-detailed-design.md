## 事实基线

1. 已验证：当 Pod 或 workload 的 label 命中规则后，Operator 会自动创建 Policy，并持续执行 reconcile。
2. 已验证：Agent 以 DaemonSet 方式运行，且节点侧 PID 解析、任务启动、NodeState 回写链路已经证明可用。
3. 已验证：数据面能力可以通过目标进程的 VmSwap 大于 0 且宿主机 Swap used 持续增长来证明。
4. 已验证：Helm 安装和升级、离线镜像导入、workload patch、验证脚本执行以及 NodeState 观测，已经构成交付与验证路径。
5. 已验证：MemAvailable 仅代表宿主机级净值，只能作为辅助参考，不能作为单次运行的硬性门禁。

## 本文档职责

1. 说明实现机制、关键时序、状态流转和验证方法。
2. 聚焦代码与运行层面的细节，不再重复需求范围或总体架构判断。
3. 为联调、排障和验收验证提供可执行的技术说明。

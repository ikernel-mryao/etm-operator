## 事实基线

以下为三份文档共享的已验证事实基线：

1. Pod/workload label -> Operator 自动创建并 reconcile Policy 已验证可用。
2. Pod 重建后可重新接管并继续管理。
3. suspend / rollback 路径已验证。
4. Agent 以 DaemonSet 方式运行，且基于 cgroup v1 的节点内 PID 解析已验证可用。
5. NodeState 写回与健康状态观测已验证。
6. delivery chain 已通过 Helm install/upgrade、离线镜像导入、workload patching、script/NodeState observation 证明。
7. 不同进程的 VmSwap 增长可以不同，这不代表失败。
8. wrapper/sleep 类进程可能出现 VmSwap=0，不能仅据此判定失败。

- control-plane loop proven
- agent/DaemonSet/node execution loop proven
- data-plane loop proven by VmSwap > 0 and Swap used growth
- delivery chain proven by Helm install/upgrade, offline image import, workload patching, script/NodeState observation
- MemAvailable is host-level net value and is auxiliary, not a single-run hard gate
